# 応樹Instagram カルーセル投稿生成アプリ（v2：カルーセル対応版）

吉澤瑛さん（人生の質向上コンサルタント）本人が、Instagramの**カルーセル投稿**を
ノーストレスで量産するためのWebアプリです。Instagram自動投稿は行いません。
**カルーセル画像PNG（8枚前後）＋投稿欄にそのまま貼れるTXT本文**をZIPでまとめてダウンロードするところまでがスコープです。

このバージョンでは、旧MVP（1枚フィード投稿＋ストーリーズ生成）を廃止し、
山添さん型TTPの「1日5回カルーセル投稿」仕様に作り替えています。ストーリーズ生成は削除しました。

---

## 1. できること

- 「生成日数」「テーマ指定（自動配分 or 固定）」「任意メモ」だけで、1日あたり **カルーセル投稿5本** を自動生成
- 1投稿＝カルーセル画像8枚（TOP／問題提起／相談／見立て／具体例／気づき／行動提案／CTA）＋投稿欄本文1本
- 画像デザインは添付のTOP／MIDDLE参考画像の方向（白ベース＋明るいライトグリーン＋上品な明朝系フォント＋余白多め）を
  Canvas上でproceduralに再現。4:5（1080×1350）比率。
- 投稿欄本文は「冒頭→よくある相談→service→Present→profile→公式LINE→ハッシュタグ」の山添さん型ブロック構成。
  service／Present／profile／よくある相談は応樹さんから素材回収中のため、設定画面で差し替え可能な仮置き文言にしています。
- 各投稿はまずOpenAI API（Netlify Functions経由・Responses API）で文章を生成し、失敗時のみローカルフレーズバンクにフォールバック
- 投稿単位の再生成・投稿欄本文の編集・投稿単位ZIP／全体ZIPダウンロード
- ZIP構成：`1日目/1投稿目/...` のように日別・投稿別フォルダ分け＋`manifest.json`
- 生成履歴の保存（ローカル）と、TOP見出し・投稿欄冒頭の重複回避への活用（設定画面からリセット可能）
- OpenAI APIキー未設定でも **ローカルのフレーズバンクで完全に動作**（保険機能）

## 2. 今回のスコープ外

- ストーリーズ生成（削除済み）
- ログイン／Supabase等クラウド保存（`src/lib/storage.ts` が将来差し替えの境界）
- 背景テンプレートの複数バリエーション管理・本人写真背景（正本デザイン1種類に統一したため削除）

```
oki-ig-mvp/
├─ index.html
├─ package.json
├─ netlify.toml
├─ .env.example
├─ netlify/functions/generate-text.ts   … OpenAI Responses APIでカルーセル1投稿分のJSONを生成
└─ src/
   ├─ App.tsx                            … タブ（投稿生成・出力確認 / 設定）
   ├─ types.ts                           … CarouselPost / Slide / AppSettings 等の型定義
   ├─ styles.css
   ├─ components/
   │  ├─ GenerateScreen.tsx              … 生成画面＋出力確認画面（日別・投稿別プレビュー）
   │  ├─ CarouselPostCard.tsx            … 1投稿分のカルーセルプレビュー・本文編集・再生成・ZIP
   │  └─ SettingsScreen.tsx              … service/Present/profile/よくある相談・LINE URL・履歴リセット等
   └─ lib/
      ├─ contentPlan.ts                  … 1日3投稿の組み立て・AI/ローカル切り替え・キャプション組立
      ├─ contentBank.ts                  … OpenAI未設定/失敗時のローカルフレーズバンク（テーマ別）
      ├─ openai.ts                       … Netlify Functions呼び出し（フロント側）
      ├─ imageGen.ts                     … カルーセルスライドをCanvasで描画（TOP/中ページ/CTA）
      ├─ zipExport.ts                    … 日別・投稿別ZIP出力＋manifest.json
      └─ storage.ts                      … 設定・履歴・生成バッチのlocalStorage保存層
```

## 4. セットアップ手順

```bash
cd oki-ig-mvp
npm install
npm run dev       # http://localhost:5173 （Netlify Functionsも使う場合は netlify dev 推奨）
```

OpenAI連携まで含めてローカル確認する場合：

```bash
npm install -g netlify-cli   # 未導入の場合のみ
netlify dev                  # http://localhost:8888
```

## 5. Netlify環境変数

Netlifyの `Site settings > Environment variables` に以下を設定してください（`.env.example` 参照）。

| 変数名 | 内容 |
|---|---|
| `OPENAI_API_KEY` | OpenAIのAPIキー（未設定でもローカルフレーズバンクで動作します） |
| `OPENAI_MODEL` | 既定モデル名（設定画面の「OpenAIモデル名」で個別に上書き可能） |

## 6. 動作確認手順（【14. 検品用テスト】準拠）

1. 「投稿生成・出力確認」タブを開く
2. 生成日数：`1`、テーマ指定：`人間関係`、任意メモ：`人と会うと疲れる人向け` を入力し「生成する」
3. 期待される結果：
   - 1日目に5投稿分のカルーセルカードが表示される
   - 各投稿に8枚前後のカルーセル画像プレビューとサムネイル一覧が表示される
   - 投稿欄本文プレビューが3本表示される（コピー・編集が可能）
   - ストーリーズは一切出力されない
4. 見た目検品：TOPは人物写真なし・白背景・明るいライトグリーン・暗い緑や灰緑がない・中央に強い一言・
   中ページはPOINT型・文字が読みやすい・フォントが混在していないことを確認
5. 「全体ZIPダウンロード」を押し、ZIP内が `1日目/1投稿目/`〜`1日目/5投稿目/` に整理され、
   各投稿フォルダに画像8枚前後＋投稿本文.txtが入っていること、ルートに`manifest.json`があることを確認

## 7. 既知の制約・今後の優先タスク

- ローカルフレーズバンクは各テーマ2バリエーションのみのため、OpenAI未設定のまま同日に同テーマで3投稿以上生成すると
  一部内容が重複します（本番運用はOpenAI APIキー設定を前提とし、バンクはあくまで保険機能です）。
- 画像デザインは添付参考画像の方向をCanvasでprocedural（アルゴリズム的）に再現したもので、
  参考画像そのものの水彩イラストを複製してはいません（著作権保護のため）。実運用でさらに近づけたい場合は、
  余白・葉の位置などのパラメータを`src/lib/imageGen.ts`で微調整してください。
- 画像比率は4:5（1080×1350）で実装済みです。


## v8 固定デザイン素材方式

この版では、Canvasで葉や水彩感を毎回描画するだけの方式から、`public/assets/design/` 配下の固定SVG背景素材を読み込んで下敷きにする方式へ切り替えています。

- `bg-top.svg`：TOPページ用の白場・水彩・植物装飾背景
- `bg-middle.svg`：中間ページ用の読み物背景
- `bg-cta.svg`：CTAページ用の案内背景

これにより、背景・葉・にじみ・紙感のブレを減らし、アプリ出力全体の高級感と統一感を上げています。
文字は従来どおりCanvas側で描画し、TOP/中面/CTAすべて濃いチャコール系の明朝体を基準にしています。


## v12 1日5回投稿仕様

標準投稿枠は以下で固定しています。

- 06:30｜健康
- 09:30｜人間関係
- 12:30｜お金
- 18:30｜ご縁
- 21:00｜使命・内省

ストーリーズ生成は引き続き行いません。

## v21 embedded approved backgrounds

v20で3・5投稿目の固定PNGが実打ち出し時に読み込まれず、fallback背景になっていたため修正。

- 3投稿目・5投稿目の承認済み背景を `src/lib/fixedBackgroundData.ts` に dataURL として埋め込み。
- 外部 `/assets/design/...` パスに依存しないため、Netlify/GitHub反映漏れや画像パス不一致で背景が消えない。
- 3・5投稿目ではCanvas追加装飾を乗せず、承認済み背景そのものを使用。


## v22 hard fixed background

背景不反映の再発防止修正。

- 2投稿目、3投稿目、5投稿目の背景をコード内dataURLに固定。
- 2投稿目：水彩の葉と金の枠（1投稿目と同系統）
- 3投稿目：春の花の葉の水彩フレーム（承認済み背景）
- 5投稿目：優雅なウォーターカラーブランチデザイン（承認済み背景）
- 2/3/5は外部画像パスに依存しない。
- 2/3/5の画像読み込みが万一失敗した場合も、Canvas直描きfallbackで背景差分を出す。
- Slide自体にも backgroundPostIndex を保持し、renderAllSlidesへのpostIndex渡し漏れにも対応。


## v23 critical fix

v22では、2・3・5投稿目の承認済み背景をdataURLで読み込んでいたが、fillBackground側の条件が `bgAsset && isFirstPost` になっていたため、1投稿目以外の背景が描画されていなかった。

v23では `bgAsset` があれば投稿番号に関係なく必ず描画するよう修正。
2・3・5投稿目の承認済み固定背景は `bgAlpha=1` で表示し、中央白かぶせも最小限にした。
