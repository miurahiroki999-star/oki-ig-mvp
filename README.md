# 応樹さん Instagram投稿生成AIアプリ（MVP）

吉澤瑛さん（人生の質向上コンサルタント）本人が、Instagramのフィード投稿とストーリーズを
ノーストレスで量産するためのWebアプリです。Instagram自動投稿は行いません。
**画像PNG＋投稿欄にそのまま貼れるTXT本文をZIPでまとめてダウンロードする**ところまでがスコープです。

---

## 1. できること（優先度A・実装済み）

- 「生成日数」と「任意メモ」だけで、1日あたり **フィード3本＋ストーリーズ5本** を自動生成
- フィード：1回目=問題提起／2回目=教育・気づき／3回目=信頼形成・導線
- ストーリーズ：1回目=問題提起／2回目=一言教育／3回目=日常人柄／4回目=信頼形成／5回目=LINE診断誘導
- 背景テンプレート＋文字合成でPNGを生成（Canvas、和文デザインフォント使用）
- 投稿本文（TXT）を自動生成、コピー・PNGダウンロード・TXTダウンロード
- 個別編集（文言だけ／画像も再生成）、個別再生成（タイトル被り・切り口被りを回避）
- ZIP一括ダウンロード（日別フォルダ＋manifest.json＋投稿一覧.txt）
- 生成履歴・再生成履歴の保存（ローカル）と、重複回避への活用
- OpenAI APIキー未設定でも **ローカルのフレーズバンクで完全に動作**（保険機能）
- 各投稿カードに「AIプロンプト」ボタン：ClaudeやChatGPTにそのまま貼れる生成用プロンプトを表示（プロンプト生成モード）

## 2. 優先度B（簡易実装）

- 背景テンプレート管理画面（追加・削除、フィード/ストーリーズ各10種類前後をプリセット）
- 写真素材集管理画面（アップロード、タグの手動選択、OpenAI Vision未設定時は手動タグにフォールバック）
- 履歴画面（検索・日付フィルター）

## 3. 優先度C（未実装・today's MVPには含めていません）

- ログイン
- Supabaseへのクラウド保存（設計は「ローカル保存→後からクラウドに差し替えやすい」構造にしてあります。`src/lib/storage.ts` がその境界です）

## 5. セットアップ手順（ローカル動作確認）

```bash
# 1. 依存関係インストール
npm install

# 2. 開発サーバー起動（OpenAI連携なしでもこのまま動きます）
npm run dev
```

ブラウザで `http://localhost:5173` を開けば、そのまま「生成する」ボタンを押して動作確認できます。
この時点では `netlify/functions` は呼ばれず、失敗時は自動的にローカルのフレーズバンクにフォールバックするため、
**OpenAI APIキーが無くても一通り動作します。**

### OpenAI連携も含めてローカル確認したい場合

Netlify CLIを使うと `netlify/functions` もローカルで動かせます。

```bash
npm install -g netlify-cli
cp .env.example .env
# .env に OPENAI_API_KEY を設定
netlify dev
```

`netlify dev` はデフォルトで `http://localhost:8888` を開き、Vite側 (`5173`) とFunctions側 (`8888`) を橋渡ししてくれます。

## 6. 環境変数

| 変数名 | 必須 | 説明 |
|---|---|---|
| `OPENAI_API_KEY` | 任意 | 設定するとNetlify Functions経由でOpenAI APIを呼び、投稿文章・写真タグ付けの精度が上がります。未設定でもアプリ自体は動作します（ローカル生成モードにフォールバック）。 |
| `OPENAI_MODEL` | 任意 | 例：`gpt-5.4-mini`。設定画面からも上書き指定できます。品質不足時は上位モデルに切り替え可能です。 |

**重要：APIキーはフロントエンドに一切置いていません。** 必ず `netlify/functions` 内（サーバー側環境変数）でのみ参照する構成です。

## 7. Netlifyへのデプロイ手順

1. このフォルダをGitHubなど任意のGitリポジトリにpush
2. Netlifyで「Add new site」→「Import an existing project」からリポジトリを選択
3. ビルド設定（`netlify.toml` に記載済みなので基本そのままでOK）
   - Build command: `npm run build`
   - Publish directory: `dist`
   - Functions directory: `netlify/functions`
4. Site settings → Environment variables で `OPENAI_API_KEY` / `OPENAI_MODEL` を設定（任意）
5. Deploy

Gitを使わない場合は、`npm run build` 後の `dist` フォルダと `netlify/functions` を
Netlifyの手動デプロイ（ドラッグ&ドロップ、ただしFunctionsを使うにはCLI経由 `netlify deploy` を推奨）でアップロードしてください。

```bash
npm run build
netlify deploy --prod
```

## 8. 使い方（応樹さん本人の日常運用）

1. 「投稿生成」画面を開く
2. 「生成日数」を入力（例：7）
3. 必要なら「自由メモ」に意図を書く（例：ご縁と瞑想を多めに）。空欄なら自動配分されます
4. 「生成する」を押す
5. 各カードでプレビュー・本文を確認し、必要なら「編集」または「再生成」
6. 問題なければ「ZIPダウンロード」→ 日別フォルダ＋PNG／TXTが入ったZIPが落ちてきます
7. Instagramアプリ側で、対応するTXTの本文をコピーして貼り付け、PNGを選んで投稿するだけです

同じ日にもう一度生成すると、自動的に「2回目打ち出し」として区別されます（ファイル名・ZIP名に反映）。

## 9. 命名ルール・出力仕様（仕様書との対応）

- ファイル名：`{打ち出し日}_{打ち出し回}回目打ち出し_{◯日目}_{◯回目}_{種別}.png / .txt`
- ZIP名：`{打ち出し日}_{打ち出し回}回目打ち出し.zip`
- ZIP内：`1日目/`, `2日目/` ... の日別フォルダにPNG・TXTを格納
- TXTの中身は投稿本文（＋自然なLINE誘導文＋ハッシュタグ）のみ。テーマや役割などの管理情報は含めず、`manifest.json` 側に分離しています
- 「◯日目」はその回の一括生成セット内での日数であり、通算日ではありません
- 朝・昼・夜の表記は使わず、すべて「◯回目」で管理しています

---

## 10. 重複回避の仕組み

- 生成のたびに `title`（画像内タイトル）／`approach`（切り口）／`claim`（主張）を履歴に保存
- 次回生成・再生成時は、直近の履歴と重複しない候補を優先的に選びます
- テーマ自体の繰り返しはOK（健康・お金・人間関係などは有限なテーマのため）という仕様書の方針に沿っています
- 「投稿済みチェック」のような手動運用は採用していません（自動履歴のみ）

---

## 11. 既知の制約（MVPとして今後拡張したい点）

- 写真素材集・履歴データは現状ブラウザのlocalStorageに保存されます（別端末・別ブラウザとは共有されません）。優先度Cの「Supabase連携」で解消予定の設計にしてあります（`src/lib/storage.ts` を差し替えるだけで移行できる構造）。
- フレーズバンクは初期セットのみ収録しています。運用しながら「切り口」のバリエーションを増やしていくと、より重複を避けやすくなります。
- OpenAI連携（`generate-text.ts` / `tag-photo.ts`）は実装済みですが、実際の文章品質はAPIキー設定後にご確認ください。
